# 사용자 흐름

## 상태 목록
- 앱 내부: 온보딩, 접근성 설정 안내, 튜토리얼, 홈, 약관 상세
- 오버레이: 대기 동그라미, 듣는 중, 찾는 중, 마스킹 안내, 스크롤 안내, 실패 안내, 민감 화면 안내

## 첫 실행
1. 온보딩 화면
   - 제목: "큐가 옆에서 알려드릴게요"
   - 문구:
     - "휴대폰 화면에서 어디를 눌러야 할지 알려드려요."
     - "큐는 대신 누르지 않아요."
     - "결제와 송금은 대신 하지 않아요."
     - "개인정보는 휴대폰 안에만 있어요."
   - 주요 버튼: "시작하기"
   - 보조 링크: "약관 보기"

2. 접근성 설정 안내
   - 제목: "화면 위에서 알려드리려면 설정이 필요해요"
   - 문구:
     - "큐는 화면의 글자와 버튼 위치를 보고 길을 알려드려요."
     - "이 권한으로 결제하거나 송금하지 않아요."
     - "계좌번호, 카드번호, 비밀번호, 인증번호는 저장하지 않아요."
   - 단계:
     - "1. 설정에서 큐를 누르세요"
     - "2. 사용을 켜세요"
   - 버튼: "설정 열기"

3. 권한 켜짐 확인
   - 켜짐: "잘 됐어요. 이제 한 번 연습해볼게요."
   - 꺼짐: "아직 켜지지 않았어요. 다시 한 번 해볼게요."

4. 튜토리얼
   - 건너뛰기 없음.
   - 마이크 권한은 실제 말하기 직전에 요청한다.
   - 단계:
     - "동그라미를 눌러보세요."
     - "무슨 도움이 필요한지 말해보세요."
     - 예시: "물티슈 찾아줘"
     - 마스킹 표시, 음성: "보이는 곳을 눌러주세요."
     - 완료: "이제 쿠팡, 카카오톡, 코레일톡에서 사용할 수 있어요."

## 평상시 사용
1. 사용자가 쿠팡/카톡/코레일톡을 직접 연다.
2. 큐 동그라미가 좌우 가장자리에 떠 있다.
3. 사용자가 앱에서 막히면 큐 동그라미를 누른다.
4. 큐가 듣는 중 패널로 바뀐다.
5. 사용자가 말한다.
6. 큐가 "찾고 있어요." 상태가 된다.
7. 결과:
   - 명확한 후보: 마스킹
   - 후보 없음 + 스크롤 가능: 스크롤 안내
   - 실패: 실패 안내
   - 민감 화면: 민감 화면 안내

## 쿠팡: 상품 찾기 해피패스
명령: "물티슈 찾아줘"

1. 큐가 쿠팡 검색 의도로 해석하고 검색어 `물티슈`를 추출한다.
2. 검색창을 찾는다.
3. 검색창만 보이게 마스킹한다.
4. 음성: "눌러주세요."
5. 사용자가 검색창을 누른다.
6. 큐가 검색어를 자동 입력한다.
7. 키보드 검색 버튼을 먼저 찾고, 없으면 앱 검색 버튼을 찾는다.
8. 검색 버튼만 보이게 마스킹한다.
9. 사용자가 검색 버튼을 누른다.
10. 큐 종료. 음성: "상품은 직접 골라주세요."

## 쿠팡: 장바구니 보기
명령: "장바구니 보고 싶어"

1. 장바구니 후보 노드를 찾는다.
2. 후보 1-3개를 노출한다.
3. 사용자가 직접 누른다.
4. 큐 종료.

## 카카오톡: 메시지 입력
명령: "아들한테 밥 먹었냐고 보내고 싶어"

1. 사람 이름과 메시지를 분리한다.
   - 사람 이름은 저장하지 않는다.
   - 메시지 본문만 메모리 세션에 둔다.
2. 큐는 채팅방을 자동 선택하지 않는다.
3. 음성: "아들 채팅방을 직접 찾아주세요. 찾으면 다음을 알려드릴게요."
4. 최대 60초 동안 채팅방 진입을 기다린다.
5. 메시지 입력칸이 보이면 채팅방으로 판단한다.
6. 입력칸만 보이게 마스킹한다.
7. 사용자가 입력칸을 누른다.
8. 큐가 메시지를 자동 입력한다.
9. 큐 종료. 음성: "보내기 전 내용을 확인해 주세요."
10. 보내기 버튼은 안내하지 않는다.

## 카카오톡: 사진 버튼 찾기
명령: "사진 보내고 싶어"

1. 현재 화면에서 사진/앨범/카메라 버튼 후보를 찾는다.
2. 후보를 노출한다.
3. 사용자가 직접 누른다.
4. 사진 선택/전송은 안내하지 않는다.

## 코레일톡: 예매한 표 보기
명령: "예매한 표 보고 싶어"

1. `승차권 확인`, `예매내역`, `나의 승차권`, `표 확인` 후보를 찾는다.
2. 후보를 노출한다.
3. 사용자가 직접 누른다.
4. 표/QR/민감 화면이 나오면 큐 종료.
5. 음성: "표 내용을 직접 확인해 주세요."

## 코레일톡: 승차권 예매 찾기
명령: "승차권 예매하고 싶어"

1. `승차권 예매`, `예매`, `기차표 예매` 후보를 찾는다.
2. 후보를 노출한다.
3. 사용자가 직접 누른다.
4. 결제/좌석 선택/개인정보 단계로 들어가면 큐 종료.

## 실패 흐름
### 음성 인식 실패
- 문구/음성: "잘 못 들었어요. 다시 말씀해 주세요."
- 버튼: "다시 말하기"
- 3회 실패 시 종료: "나중에 다시 도와드릴게요."

### 지원하지 않는 앱
- 문구/음성: "지금은 쿠팡, 카카오톡, 코레일톡에서 도와드릴 수 있어요."
- 버튼: "다시 말하기"

### 지원하지 않는 요청
- 문구/음성: "아직 이 도움은 준비 중이에요."
- 지원 범위 2-3개를 짧게 보여준다.
- 버튼: "다시 말하기"

### 버튼을 못 찾음
- 스크롤 가능하면 마스킹 없이 큰 아래 화살표와 음성 "아래로 밀어주세요."
- 최대 3회 재탐색.
- 이후 실패: "지금 화면에서는 찾지 못했어요."
- 버튼: "다시 말하기"

### 후보가 너무 많음
- "비슷한 곳이 많아서 정확히 고르기 어려워요."
- 버튼: "다시 말하기"

### 민감 화면
- "개인정보가 있는 화면이에요. 직접 확인해 주세요."
- 선택: "입력칸 위치만 알려드릴까요?"
- 결제/송금/주문 확정/보내기 버튼은 안내하지 않는다.

## 홈 화면
정상:
- "큐가 켜져 있어요"
- "화면 옆 동그라미를 누르고 말해보세요."
- 지원 앱: 쿠팡, 카카오톡, 코레일톡
- 버튼: "튜토리얼 다시 보기"

동그라미 꺼짐:
- "큐 동그라미가 꺼져 있어요"
- 버튼: "다시 켜기"

권한 꺼짐:
- "큐를 사용하려면 설정을 켜야 해요"
- 버튼: "설정 열기"
